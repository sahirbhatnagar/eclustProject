Eclust Vignette

In this vignette we present the main functions in the eclust package used to fit a penalization model that simultaneous performs variable selection and estimation, while preserving the strong heredity principle, i.e., the interaction terms can only be included if the main effects are in the model.

Load Required Packages

library(eclust)

Simulate Data

set.seed(123456)

# number of predictors
p = 10 

# number of test subjects
n = 200

# correlation between X's
rho = 0.50

# signal to noise ratio
signal_to_noise_ratio = 4

# names of the main effects, this will be used in many of the functions
main_effect_names <- paste0("x",1:p) 

# names of the active set
true_var_names <- c("x1","x2","x3","x4","x1:x2", "x1:x3", "x1:x4", "x2:x3", "x2:x4", "x3:x4")

# different true coefficient vectors as in Table 1 of Choi et al. 
beta1 <- c(7,2,1,1,0,0,0,0,0,0) %>% magrittr::set_names(true_var_names)
beta2 <- c(7,2,1,1,1,0,0,0.5,0.4,0.1) %>% magrittr::set_names(true_var_names)
beta3 <- c(7,2,1,1,7,7,7,2,2,1) %>% magrittr::set_names(true_var_names)
beta4 <- c(7,2,1,1,14,14,14,4,4,2) %>% magrittr::set_names(true_var_names)
beta5 <- c(0,0,0,0,7,7,7,2,2,1) %>% magrittr::set_names(true_var_names)

# simulate Toeplitz like correlation structure between X's
H <- abs(outer(1:p, 1:p, "-"))
cor <- rho^H

# generate X's from multivariate normal and label the matrix
DT <- MASS::mvrnorm(n = n, mu = rep(0,p), Sigma = cor) %>% 
    magrittr::set_colnames(paste0("x",1:p)) %>% 
    set_rownames(paste0("Subject",1:n))

# create X matrix which contains all main effects and interactions
# but not the intercept
X <- model.matrix(
    as.formula(paste0("~(",paste0(main_effect_names, collapse = "+"),")^2-1")), 
    data = DT %>% as.data.frame()) 

# generate response with user defined signal to noise ratio 
y.star <- X[,names(beta4)] %*% beta4
error <- rnorm(n)
k <- sqrt(var(y.star)/(signal_to_noise_ratio*var(error))) 
Y <- y.star + k*error 
colnames(Y) <- "Y"

# names of interaction variables assuming interaction terms contain a ":"
# this will be used in many of the functions
# names must appear in the same order as X matrix
interaction_names <- grep(":", colnames(X), value = T)
main_effect_names <- setdiff(colnames(X), interaction_names)

Analysis

The main function for fitting the strong heredity interaction model is the shim function. The function has many options, but most have sensible defaults. We show all the arguments here for completeness. The required user inputs are the data, the main effects variable names and the interaction variable names. It is recommended to let the shim chose the grid of tuning paramters for you which is the default (lambda.beta = NULL and lambda.gamma = NULL)

Type ?shim in R for full documentation and description of the arguments. 

res <- shim(x = X, y = Y,
            main.effect.names = main_effect_names,
            interaction.names = interaction_names,
            lambda.beta = NULL,
            lambda.gamma = NULL, 
            nlambda.gamma = 12, 
            nlambda.beta = 10,
            nlambda = 120, 
            threshold = 1e-04, 
            max.iter = 200,
            initialization.type = "ridge", 
            intercept = TRUE,
            normalize = TRUE, 
            verbose = TRUE, 
            cores = 2)

If verbose=TRUE then the function will print the progress of the convergence of the algorithm. The binary vectors of 0's and 1's represent whether the algorithm has converged for a given pair of tuning parameters.

Cross Validation using the cv.shim function

To determine the optimal pair of tuning parameters, we perform K-fold cross validation. By default k=10. To speed up computations, the user has the option to register a parallel backend to perform the analysis on each of the folds on a different core using the doMC package:

library(doMC)
registerDoMC(cores = 10)
cv.res <- cv.shim(x = X, y = Y,
            main.effect.names = main_effect_names,
            interaction.names = interaction_names,
            lambda.beta = NULL,
            lambda.gamma = NULL, 
            nlambda.gamma = 12, 
            nlambda.beta = 10,
            nlambda = 120, 
            threshold = 1e-04, 
            max.iter = 200,
            initialization.type = "ridge", 
            intercept = TRUE,
            normalize = TRUE, 
            verbose = FALSE, 
            parallel = TRUE,
            type.measure = c("mse"), 
            nfolds = 10)

Cross Validation Plot

To visualize the CV curves, we can use the plot method for objects of class cv.shim as follows:

plot(cv.res)

The above figure shows the CV curves for each pair of tuning parameters on the log scale. The top row of numbers in each figure represents the number of non-zero main effect coefficients, and the numbers on the bottom row represent the number of non-zero interaction coefficients. Furthermore, two solid vertical lines are drawn at the minimum CV error, as well as at the tuning parameters that give the most regularised model such that the error is within one standard error of the minimum.
